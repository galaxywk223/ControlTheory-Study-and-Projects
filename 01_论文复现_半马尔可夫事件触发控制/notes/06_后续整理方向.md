# 后续整理方向

## 1. 结果对照

### 1.1 补论文结果对照表

可以整理一张明确的对照表，至少包含：

- 论文中的案例或图号；
- 对应脚本；
- 使用参数；
- 当前结果；
- 与论文是否一致。

这是把“可运行原型”提升为“可核对复现”的关键一步。

### 1.2 切分主线与扩展线

目前仓库同时包含：

- 原论文主线：半马尔可夫切换 + 扰动可测；
- 扩展线：马尔可夫跳变 + 扰动观测器。

可以继续明确：

- 哪些脚本属于主线复现；
- 哪些脚本属于扩展实现；
- 哪些结果图对应主线，哪些只是扩展尝试。

## 2. 记号与命名统一

### 2.1 统一记号和命名

后续可统一以下内容：

- $\zeta(t)$ / $r(t)$；
- $\omega(t)$ / $w(t)$；
- $\Lambda_i$ / $\Phi_i$；
- 理论变量名 / MATLAB 变量名 / Python 变量名。

### 2.2 统一脚本命名

当前 MATLAB 主线脚本仍保留 `zIE_*` 命名，如需增强可读性，可以改成更直观的名称，例如：

- `01_setup_system.m`
- `02_solve_controller.m`
- `03_solve_observer.m`
- `04_solve_trigger_matrix.m`
- `05_simulate_without_hatw.m`
- `06_simulate_with_hatw.m`
- `07_sweep_sigma.m`

## 3. 结果统计与运行说明

### 3.1 增加结果统计

可以补充以下统计量：

- 触发次数；
- 平均事件间隔；
- 最小/最大事件间隔；
- 状态峰值；
- 收敛速度。

### 3.2 增加运行说明

可以补充以下文档：

- Python 环境依赖；
- MATLAB 工具箱依赖；
- 运行顺序；
- 输出文件说明。

## 4. 推荐整理顺序

沿当前结构继续推进时，可以按下面顺序整理：

1. 先补“论文结果对照表”；
2. 再统一脚本命名和理论记号；
3. 最后补环境说明和结果统计表。
